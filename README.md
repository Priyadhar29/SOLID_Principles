## üìò Overview

The **SOLID Principles** are the foundation of writing clean, maintainable, and scalable Object-Oriented code.  
Each principle focuses on improving software design, reducing coupling, and making your codebase easier to understand and extend.

## üß© The Five SOLID Principles

| Principle | Full Form | Focus | Example Topic |
|------------|------------|--------|----------------|
| **S** | Single Responsibility Principle | One class = One job | E-commerce order management |
| **O** | Open/Closed Principle | Open for extension, closed for modification | Payment gateway system |
| **L** | Liskov Substitution Principle | Subtypes must be replaceable for base types | Vehicle inheritance |
| **I** | Interface Segregation Principle | Clients shouldn‚Äôt depend on unused methods | Smart device interfaces |
| **D** | Dependency Inversion Principle | Depend on abstractions, not on concretes | Notification service with dependency injection |
